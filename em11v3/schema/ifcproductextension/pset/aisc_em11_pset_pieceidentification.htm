5.2.4.3 AISC_EM11_Pset_PieceIdentification

Piece Identification: A piece can have identification information

buildingSMART Data Dictionary

PSD-XML

NameTypeDescription
PieceMark P_SINGLEVALUE / IfcIdentifier
Piece MarkA short text reference for the piece. A means of identifying the piece within the fabrication shop
IndicationMark P_SINGLEVALUE / IfcIdentifier
Indication MarkMark for indication on part (may differ from Piece or assembly mark)
MainPieceTag P_SINGLEVALUE / IfcBoolean
Main Piece TagIndicates if it is main piece or not
PrelimMark P_SINGLEVALUE / IfcIdentifier
Prelim MarkA short text reference that may be used to identify the located_part (piece) at the bidding or tender stages of its production life.
BarCode P_SINGLEVALUE / IfcIdentifier
BarcodeA numerical interpretaion of the bar code associated with the located_part (piece). The bar code is sometime used to identify the piece in the fabrication shop, during transportation, and during erection using a bar code scanner.
CustomType P_SINGLEVALUE / IfcBoolean
Custom TypeIndicate if the piece has custom profile that can be either user defined custom profile such as I type, or custom multi plates piece. This property is only needed if it is a custom piece and the value should be 'true'. When representing a custom multi plates piece, either individual piece can be modeled as a geometrical object or as a plate object. Representing as a collection of geometrical objects should only be used if receiving application does not need to process individual pieces, such as use of bought out item. Representing as a collection of plate object should be should in all other cases when fabrication information of each component of the custom piece is required.
